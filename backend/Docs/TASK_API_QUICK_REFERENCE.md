# Task API - Quick Reference

## Summary of All Endpoints

| Method | Endpoint | Description | Auth | Status |
|--------|----------|-------------|------|--------|
| GET | /api/tasks | Get all tasks | Required | 200 |
| GET | /api/tasks/{id} | Get task by ID | Required | 200 |
| POST | /api/tasks | Create task | Required | 201 |
| PUT | /api/tasks/{id} | Update task (full) | Required | 200 |
| PATCH | /api/tasks/{id} | Update task (partial) | Required | 200 |
| DELETE | /api/tasks/{id} | Delete task | Required | 204 |

---

## Field Reference

### Task Fields

| Field | Type | Create | Update | Patch | Notes |
|-------|------|--------|--------|-------|-------|
| id | Long | Auto | Auto | Read-only | Generated by database |
| title | String | Required | Required | Optional | 1-255 characters |
| description | String | Optional | Required | Optional | Max 2000 characters |
| status | String | Optional | Required | Optional | TODO, IN_PROGRESS, COMPLETED |
| userId | Long | Auto | Auto | Read-only | From JWT token |
| userEmail | String | Auto | Auto | Read-only | From JWT token |
| createdAt | DateTime | Auto | Auto | Read-only | Never changes |
| updatedAt | DateTime | Auto | Auto | Read-only | Updated on every change |

---

## Status Values

```
TODO           → Task not yet started
IN_PROGRESS    → Task is being worked on
COMPLETED      → Task is finished
```

---

## Example Requests

### 1. Get All Tasks
```bash
GET /api/tasks
Authorization: Bearer <token>
```

### 2. Get Specific Task
```bash
GET /api/tasks/1
Authorization: Bearer <token>
```

### 3. Create Task
```bash
POST /api/tasks
Authorization: Bearer <token>
Content-Type: application/json

{
  "title": "Task title",
  "description": "Optional description",
  "status": "TODO"
}
```

### 4. Update Task (Full)
```bash
PUT /api/tasks/1
Authorization: Bearer <token>
Content-Type: application/json

{
  "title": "Updated title",
  "description": "Updated description",
  "status": "IN_PROGRESS"
}
```

### 5. Update Task (Partial)
```bash
PATCH /api/tasks/1
Authorization: Bearer <token>
Content-Type: application/json

{
  "status": "COMPLETED"
}
```

### 6. Delete Task
```bash
DELETE /api/tasks/1
Authorization: Bearer <token>
```

## Response Codes

| Code | Meaning |
|------|---------|
| 200 | Success - Request completed |
| 201 | Created - New resource created |
| 204 | No Content - Deleted successfully |
| 400 | Bad Request - Invalid input |
| 401 | Unauthorized - Invalid/missing token |
| 404 | Not Found - Resource doesn't exist |
| 500 | Server Error - Internal error |

## PUT vs PATCH

| Aspect | PUT | PATCH |
|--------|-----|-------|
| Full/Partial | Full update | Partial update |
| Required fields | All required | All optional |
| Use case | Replace entire resource | Update single field |
| Data efficiency | More data | Less data |

## Authentication

All endpoints require JWT token in Authorization header:

```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

Get token from login endpoint:
```bash
POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "password123"
}
```

## Postman Collection Template

Save as environment variables in Postman:

```
{{base_url}} = http://localhost:8080
{{token}} = (JWT token from login)
{{task_id}} = (task ID to test)
```

Then use in requests:

```
Authorization: Bearer {{token}}
GET {{base_url}}/api/tasks
GET {{base_url}}/api/tasks/{{task_id}}
POST {{base_url}}/api/tasks
```

## cURL Cheat Sheet

```bash
# Set token variable
TOKEN="your_jwt_token_here"

# Get all tasks
curl -X GET http://localhost:8080/api/tasks \
  -H "Authorization: Bearer $TOKEN"

# Create task
curl -X POST http://localhost:8080/api/tasks \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"Task","description":"Desc","status":"TODO"}'

# Get task by ID
curl -X GET http://localhost:8080/api/tasks/1 \
  -H "Authorization: Bearer $TOKEN"

# Update task (PUT)
curl -X PUT http://localhost:8080/api/tasks/1 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"New","description":"New","status":"IN_PROGRESS"}'

# Partial update (PATCH)
curl -X PATCH http://localhost:8080/api/tasks/1 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status":"COMPLETED"}'

# Delete task
curl -X DELETE http://localhost:8080/api/tasks/1 \
  -H "Authorization: Bearer $TOKEN"
```

## Key Points to Remember

1. ✅ **Authentication Required** - Always include valid JWT token
2. ✅ **User Isolation** - You can only access your own tasks
3. ✅ **Status Values** - Must be TODO, IN_PROGRESS, or COMPLETED
4. ✅ **PUT vs PATCH** - PUT needs all fields, PATCH needs only changed fields
5. ✅ **Timestamps** - createdAt never changes, updatedAt updates on every change
6. ✅ **Validation** - Title required (1-255), description optional (max 2000)
