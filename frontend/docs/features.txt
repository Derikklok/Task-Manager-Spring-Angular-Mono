🎯 Overview
This is the backend API for a Task Manager application built with Spring Boot 4.0.3 and Java 21. It provides:

User Authentication - Secure registration and login with JWT tokens
Task Management - Full CRUD operations with user isolation
Health Monitoring - Public health check endpoints for Docker and Kubernetes
Docker Integration - Multi-stage Dockerfile and Docker Compose support
Production Ready - Validation, error handling, security, and best practices
Status: ✅ Production Ready | Version: 0.0.1-SNAPSHOT | Date: March 18, 2026

✨ Key Features
🔐 Authentication & Authorization
User registration with email validation
Secure login with BCrypt password hashing
JWT token generation (24-hour expiration)
Role-based access control (MANAGER, DEVELOPER)
Automatic user association from JWT token
📝 Task Management
Create, Read, Update, Delete tasks
Full update (PUT) and Partial update (PATCH)
User isolation (each user sees only their tasks)
Automatic timestamps (createdAt, updatedAt)

📍 API Endpoints
✅ Health Check Endpoints (Public - No Auth Required)
Method	Endpoint	Description	Response
GET	/api/health	Simple health check	OK
GET	/api/health/detailed	Detailed health info	JSON
GET	/api/health/live	Kubernetes liveness probe	LIVE
GET	/api/health/ready	Kubernetes readiness probe	READY
Test Health:

curl http://localhost:8080/api/health
# Response: OK
🔐 Authentication Endpoints
Method	Endpoint	Description	Body
POST	/api/auth/register	Register new user	{email, password, role}
POST	/api/auth/login	Login & get JWT token	{email, password}
Register:

curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"password123","role":"DEVELOPER"}'
Login:

curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"password123"}'
# Returns JWT token
📝 Task Management Endpoints (Protected - JWT Required)
Method	Endpoint	Description
GET	/api/tasks	Get all tasks (current user)
GET	/api/tasks/{id}	Get specific task by ID
POST	/api/tasks	Create new task
PUT	/api/tasks/{id}	Update task (full update)
PATCH	/api/tasks/{id}	Update task (partial update)
DELETE	/api/tasks/{id}	Delete task
All protected endpoints require JWT token:

-H "Authorization: Bearer <JWT_TOKEN>"
Create Task:

curl -X POST http://localhost:8080/api/tasks \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"title":"My Task","description":"Description","status":"TODO"}'
Get All Tasks:

curl -X GET http://localhost:8080/api/tasks \
  -H "Authorization: Bearer <TOKEN>"
Partial Update (PATCH):

curl -X PATCH http://localhost:8080/api/tasks/1 \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"status":"COMPLETED"}'
Delete Task:

curl -X DELETE http://localhost:8080/api/tasks/1 \
  -H "Authorization: Bearer <TOKEN>"

  -p 8080:8080 \
  -e DB_URL=jdbc:mysql://mysql_host:3306/task_manager \
  -e DB_USERNAME=root \
